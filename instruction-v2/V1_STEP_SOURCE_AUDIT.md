# V1 Step Source Audit

Created: 2026-06-03

## Executive Finding

The actual V1 source for crop step numbers is not one single raw detector output.

For fresh generation, V1 uses `clean/routers/instruction_debug.py::_build_instruction_callout_crops`, which combines:

- bag ranges from V1 bag truth,
- `clean/services/step_detector_service.py::detect_steps`,
- `clean/routers/instruction_debug.py::_filter_invalid_step_anchor_boxes`,
- page-level callout/step fallback functions,
- callout crop detection,
- qty OCR validation.

For already-proven V1 output, the authoritative evidence is persisted in:

- `debug/crop_cache/{set_num}_bag{bag}.json`
- `debug/training_labels/{set_num}_bag{bag}.json`
- `debug/ai_training/analysis_bundles/.../metadata.json`

This matters because current raw `step_detector_service.detect_steps` does **not** reproduce every remembered working page. Page 7 is the clearest example: raw `detect_steps` returns no main steps, but V1 persisted crop output contains `p7_s1_c1` and `p7_s2_c2`.

## Traced Sources

### Crop builder

Source:

- `clean/routers/instruction_debug.py::_build_instruction_callout_crops`

Key behavior:

- Resolves bag page range with `_resolve_bag_page_range`.
- Calls `step_detector_service.detect_steps`.
- Converts detector output with `_contact_sheet_step_boxes_from_detected`.
- Filters bad anchors with `_filter_invalid_step_anchor_boxes`.
- Optionally recovers missing right-half steps with local OCR.
- Detects callout boxes using `_detect_callout_rect_by_edges`.
- Falls back to `_page_level_callout_candidates_for_fallback`.
- Extracts qty via `_qty_payload_for_page_level_callout_crop` / `_auto_qty_payload_for_crop`.
- Emits crop records with `page`, `step`, `qty_text`, `qty_numbers`, `crop_box`, and `crop_id`.

Crop ID generation line:

```text
crop_id = f"p{int(page)}_s{max(step_number, 0)}_c{idx}"
```

### Step detector

Source:

- `clean/services/step_detector_service.py::detect_steps`

Output fields:

- `step_candidates`
- `classified_step_boxes`
- `main_step`
- `main_steps`
- `sub_steps`
- `numeric_tokens`
- `joined_numbers`
- `visual_candidates`

Current detector proof:

| Page | Raw `detect_steps` result |
| ---: | --- |
| 7 | no main steps, no classified boxes |
| 12 | main steps 11 and 12 |
| 23 | main step 27 |
| 81 | no main steps, no classified boxes |

### Step anchor filter

Source:

- `clean/routers/instruction_debug.py::_filter_invalid_step_anchor_boxes`

Behavior:

- Keeps step numbers where `0 < step_number < 1000`.
- Rejects impossible OCR anchors like 4-digit numbers.

This is applied inside `_build_instruction_callout_crops` after `detect_steps`.

### Page-level fallback step detection

Sources:

- `clean/routers/instruction_debug.py::_detect_page_step_number_boxes`
- `clean/routers/instruction_debug.py::_detect_step_number_below_panel`
- `clean/routers/instruction_debug.py::_page_level_callout_candidates_for_fallback`

Behavior:

- Uses step boxes when available.
- If no step boxes exist, tries page OCR for step numbers.
- If page-level callout panels exist, can OCR step numbers below panels.
- Assigns panels to detected step numbers.

This is likely how persisted page 7 crops were produced, because raw `step_detector_service.detect_steps` currently returns no step boxes for page 7, while persisted V1 crop output contains steps 1 and 2.

## Answers

### 1. How is `p23_s27_c1` generated?

`p23_s27_c1` is generated by:

- `clean/routers/instruction_debug.py::_build_instruction_callout_crops`

The generation format is:

```text
p{page}_s{step_number}_c{idx}
```

For `p23_s27_c1`:

- page = 23
- step_number = 27
- callout candidate index = 1

Persisted proof:

- `debug/crop_cache/70618_bag2.json`
- `crop_id: p23_s27_c1`
- `page: 23`
- `step: 27`
- `source: edge_detect`
- `crop_box: [114, 29, 313, 161]`

### 2. What is the authoritative source for `step_number`?

For fresh V1 crop generation, the authoritative source is the `step_number` attached to each callout candidate inside `_build_instruction_callout_crops`.

That `step_number` may come from:

- `step_detector_service.detect_steps` via classified main step boxes,
- filtered by `_filter_invalid_step_anchor_boxes`,
- right-half gap recovery OCR inside `_build_instruction_callout_crops`,
- page-level fallback functions such as `_detect_page_step_number_boxes` and `_detect_step_number_below_panel`.

For already-proven V1 data, the authoritative source is persisted crop output:

- `debug/crop_cache/{set_num}_bag{bag}.json`
- then `debug/training_labels/{set_num}_bag{bag}.json` for reviewed/saved label state.

### 3. Is `step_number` detector-only or persisted?

It is both, depending on phase.

Fresh generation:

- detector/fallback-derived inside `_build_instruction_callout_crops`.

Persisted V1 output:

- saved in `debug/crop_cache/...` as `step`,
- copied/saved in `debug/training_labels/...` as `step`,
- embedded in analysis bundle IDs such as `70618_bag2_p23_s27_c1`.

So V2 should not assume raw `detect_steps` alone is authoritative.

### 4. Which V1 file/service should V2 import next?

Next V2 recovery should import/read the persisted V1 crop cache before regenerating steps:

```text
debug/crop_cache/{set_num}_bag{bag}.json
```

Then V2 should map it into a V2 manifest, likely before or replacing fresh Stage 5/6 output:

```text
05/06 V1 crop truth import or 06_v1_callout_crop_import.json
```

If V2 must regenerate rather than import, the exact V1 source to port is:

1. `clean/routers/instruction_debug.py::_build_instruction_callout_crops`
2. `clean/services/step_detector_service.py::detect_steps`
3. `clean/routers/instruction_debug.py::_filter_invalid_step_anchor_boxes`
4. `clean/routers/instruction_debug.py::_page_level_callout_candidates_for_fallback`
5. `clean/routers/instruction_debug.py::_detect_page_step_number_boxes`
6. `clean/routers/instruction_debug.py::_detect_step_number_below_panel`

But the immediate recovery item should be a read-only V1 crop-cache import, because it preserves the already-working V1 step/crop decisions.

## Recommendation

Do not port raw step detection as the next recovery item.

Next item should be:

```text
V1 crop cache import only
Read debug/crop_cache/{set_num}_bag{bag}.json
Write a V2 manifest preserving crop_id, page, step, qty, source, crop_box
```

That gives V2 the actual working V1 step/crop truth before any fresh regeneration attempts.
